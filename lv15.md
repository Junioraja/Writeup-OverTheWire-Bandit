## Deskripsi singkat

Di level ini saya harus mengirim password level saat ini ke service yang berjalan di localhost pada port 30001  
Berbeda dengan level sebelumnya service ini menggunakan koneksi ssl tls  
Artinya saya tidak bisa menggunakan netcat biasa dan perlu menggunakan openssl  

---

## 1. Login ke server

Saya login sebagai bandit15 menggunakan password dari level sebelumnya  

```bash
ssh bandit15@bandit.labs.overthewire.org -p 2220
```

Setelah berhasil masuk saya siap berinteraksi dengan service  

---

## 2. Terhubung ke service menggunakan ssl

Karena service menggunakan ssl saya menggunakan perintah berikut  

```bash
openssl s_client -connect localhost:30001
```

Setelah koneksi berhasil dibuat saya menempelkan password level 15 lalu menekan enter  

---

## 3. Mendapatkan password baru

Jika password yang saya kirim benar maka server akan merespons dengan string baru  

String tersebut adalah password untuk bandit16  

---

## Kesimpulan

Di level ini saya belajar bahwa tidak semua service berjalan dengan koneksi biasa  
Beberapa menggunakan ssl tls sehingga perlu tool yang sesuai seperti openssl s_client  
Level ini melatih pemahaman tentang perbedaan koneksi tcp biasa dan koneksi terenkripsi
