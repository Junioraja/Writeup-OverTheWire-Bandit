## Tujuan

## Deskripsi singkat

Di level ini terdapat sebuah file executable bernama bandit20-do  
File ini dapat dijalankan untuk mengeksekusi perintah sebagai user bandit20  
Artinya file tersebut memiliki setuid dan memungkinkan saya membaca file milik bandit20  

---

## 1. Login ke server

Saya login sebagai bandit19 menggunakan password dari level sebelumnya  

```bash
ssh bandit19@bandit.labs.overthewire.org -p 2220
```

Setelah berhasil masuk saya melihat isi direktori  

```bash
ls -la
```

Terlihat file executable bernama bandit20-do  

---

## 2. Memahami fungsi file executable

Saya mencoba menjalankan file tersebut tanpa argumen  

```bash
./bandit20-do
```

Output menunjukkan bahwa saya harus memberikan perintah sebagai argumen  

Artinya saya bisa menjalankan perintah sebagai user bandit20  

---

## 3. Membaca password level berikutnya

Karena password biasanya disimpan di /etc/bandit_pass saya menjalankan  

```bash
./bandit20-do cat /etc/bandit_pass/bandit20
```

Perintah tersebut menampilkan string panjang  

String tersebut adalah password untuk bandit20  

---

## Kesimpulan

Di level ini saya belajar tentang file setuid yang dapat menjalankan perintah sebagai user lain  
Saya memahami bagaimana privilege escalation sederhana bisa terjadi melalui binary tertentu  
Level ini melatih pemahaman tentang permission dan eksekusi program di linux
