## Tujuan

Mencari password untuk level 2.

## Deskripsi Singkat

Di level ini, password untuk level berikutnya disimpan dalam sebuah file bernama `-`
Awalnya terlihat mudah, tapi ternyata nama file tersebut bentrok dengan cara kerja command di linux

---

## 1. Login ke Server

Pertama, saya login menggunakan password yang didapat dari level sebelumnya:

```bash
ssh bandit1@bandit.labs.overthewire.org -p 2220
```

Setelah itu saya masukkan password level 1 dan berhasil masuk ke server

---

## 2. Mengecek Isi Direktori

Saya cek isi direktori dengan:

```bash
ls
```

Output yang muncul:

```
-
```

Ternyata hanya ada satu file dengan nama `-`

---

## 3. Percobaan Awal

Awalnya saya mencoba membaca file tersebut dengan:

```bash
cat -
```

Tapi tidak menampilkan password
Saya sadar kalau di linux, tanda `-` biasanya dianggap sebagai stdin (standard input), bukan nama file biasa

---

## 4. Cara Mengakses File dengan Nama `-`

Karena `-` dianggap sebagai flag, saya perlu memberi tahu sistem bahwa itu benar-benar nama file

Saya menggunakan:

```bash
cat ./-
```

Setelah itu barulah muncul string panjang yang merupakan password untuk `bandit2`.

Alternatif lain yang juga bisa digunakan:

```bash
cat -- -
```

---

## Kesimpulan

Di level ini saya belajar bahwa nama file bisa saja bentrok dengan opsi command di linux
Untuk mengatasinya, bisa menggunakan `./` agar dianggap sebagai file di direktori saat ini, atau memakai `--` untuk menghentikan parsing opsi

Level ini melatih pemahaman dasar tentang bagaimana Linux memproses argumen pada command
