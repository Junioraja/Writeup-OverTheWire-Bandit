## Deskripsi singkat

Di level ini password untuk level berikutnya disimpan dalam file yang memiliki spasi pada namanya  
Hal ini membuat file tidak bisa diakses dengan cara biasa karena spasi dianggap sebagai pemisah argumen di linux

---

## 1 Login ke server

Saya login menggunakan password yang didapat dari level sebelumnya

```bash
ssh bandit2@bandit.labs.overthewire.org -p 2220
```

Setelah memasukkan password level 2 saya berhasil masuk ke sistem

---

## 2 Mengecek isi direktori

Saya melihat isi direktori dengan perintah

```bash
ls
```

Output yang muncul

```
spaces in this filename
```

Terlihat bahwa nama file tersebut mengandung beberapa spasi

---

## 3 Percobaan awal

Saya mencoba membaca file dengan cara biasa

```bash
cat spaces in this filename
```

Namun perintah tersebut gagal karena linux menganggap setiap kata sebagai argumen yang berbeda

---

## 4 Cara mengakses file dengan spasi

Agar bisa dibaca saya perlu memperlakukan seluruh nama file sebagai satu kesatuan  

Saya menggunakan tanda kutip

```bash
cat "spaces in this filename"
```

Setelah itu muncul string panjang yang merupakan password untuk bandit3

Alternatif lain yang juga bisa digunakan adalah dengan escape character

```bash
cat spaces\ in\ this\ filename
```

---

## Kesimpulan

Di level ini saya belajar bahwa spasi dalam nama file harus ditangani dengan benar  
Bisa menggunakan tanda kutip atau escape character agar linux membaca seluruh teks sebagai satu nama file  
Level ini melatih pemahaman tentang bagaimana shell memproses argumen
