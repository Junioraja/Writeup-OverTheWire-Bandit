## Deskripsi singkat

Di level ini saya tidak mendapatkan password dalam bentuk teks seperti biasanya  
Sebagai gantinya saya diberikan private key untuk login ke level berikutnya  
Artinya saya harus menggunakan autentikasi ssh berbasis key bukan password  

---

## 1 Login ke server

Saya login seperti biasa menggunakan password dari level sebelumnya  

```bash
ssh bandit13@bandit.labs.overthewire.org -p 2220
```

Setelah berhasil masuk saya melihat isi direktori  

```bash
ls
```

Terlihat file bernama sshkey.private  

---

## 2 Menggunakan private key untuk login

File tersebut adalah private key untuk user bandit14  
Saya gunakan key tersebut untuk login ke level berikutnya  

```bash
ssh -i sshkey.private bandit14@localhost -p 2220
```

Karena saya sudah berada di server yang sama saya menggunakan localhost  

Jika muncul konfirmasi saya ketik yes  

Setelah itu saya berhasil login sebagai bandit14 tanpa memasukkan password  

---

## 3 Mengambil password level berikutnya

Setelah login sebagai bandit14 saya melihat isi direktori  

```bash
ls
```

Terdapat file bernama readme  

Saya membaca file tersebut  

```bash
cat readme
```

Output yang muncul adalah string yang merupakan password untuk bandit14 yang akan digunakan pada level selanjutnya  

---

## Kesimpulan

Di level ini saya belajar menggunakan ssh private key untuk autentikasi  
Saya memahami perbedaan antara login menggunakan password dan login menggunakan key  
Level ini melatih pemahaman tentang mekanisme autentikasi ssh di linux
