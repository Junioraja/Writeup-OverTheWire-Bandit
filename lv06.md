## Deskripsi singkat

Di level ini password disimpan di suatu file yang berada di sistem  
File tersebut memiliki kriteria tertentu yaitu dimiliki oleh user bandit7 dimiliki oleh group bandit6 dan berukuran 33 bytes  
Lokasinya tidak berada di folder home sehingga perlu melakukan pencarian di seluruh sistem  

---

## 1. Login ke server

Saya login menggunakan password dari level sebelumnya  

```bash
ssh bandit6@bandit.labs.overthewire.org -p 2220
```

Setelah berhasil masuk saya berada di home directory  

---

## 2. Mencari file sesuai kriteria

Karena file tidak berada di direktori saat ini saya menggunakan perintah find dari root  

```bash
find / -type f -user bandit7 -group bandit6 -size 33c 2>/dev/null
```

Penjelasan singkat  
/type f untuk mencari file biasa  
-user bandit7 untuk mencari file milik user bandit7  
-group bandit6 untuk mencari file dengan group bandit6  
-size 33c untuk mencari file berukuran 33 bytes  
2>/dev/null untuk menyembunyikan pesan error permission denied  

Dari hasil tersebut muncul satu lokasi file yang sesuai  

---

## 3. Membaca isi file

Setelah mengetahui path file saya membaca isinya dengan  

```bash
cat /path/ke/file
```

Kemudian muncul string yang merupakan password untuk bandit7  

---

## Kesimpulan

Di level ini saya belajar menggunakan find dengan kombinasi beberapa filter sekaligus  
Saya juga belajar melakukan pencarian dari root directory serta menghilangkan error output dengan redirect ke /dev/null  
Level ini melatih pemahaman tentang permission dan kepemilikan file di linux
